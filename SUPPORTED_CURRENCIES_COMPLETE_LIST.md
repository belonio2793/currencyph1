## How This Works in the Application

### User Perspective
1. User goes to **Wallet Page** (/wallets)
2. Clicks **"Add More Currencies"** button
3. Modal opens with:
   - **Fiat Currency Tab** (25 currencies, blue UI)
   - **Cryptocurrency Tab** (40+ currencies, orange UI)
4. User selects desired currencies
5. System creates wallets for all selected currencies
6. **Wallet Type** is automatically set based on currency type
7. **Exchange rates** are fetched from:
   - OpenExchangeRates API (fiat currencies)
   - CoinGecko API (cryptocurrencies)
   - Fallback to cached rates if APIs unavailable

### Database Level
- Each currency is stored in `currencies` table with its `type` (fiat or crypto)
- When a wallet is created, its `type` is automatically populated from the currency type
- Wallets can be filtered by type in queries: `WHERE type = 'crypto'`
- Exchange rates stored in `pairs` table with from_currency and to_currency

### Technical Details
- **Fiat Conversion:** USD is base, all fiat pairs relative to USD
- **Crypto Pricing:** Available in multiple currencies (USD, PHP, etc.)
- **Precision:** Fiat has 2-0 decimals, Crypto has 8 decimals
- **Updates:** Rates cached for 1 hour, refreshed via edge function

## Adding New Currencies

If you want to add more currencies:

1. **Add to `currencies` table:**
```sql
INSERT INTO public.currencies (code, name, type, symbol, decimals, active)
VALUES ('NEW', 'New Currency', 'crypto', 'NEW', 8, true);
```

2. **UI automatically updates** - Modal will show the new currency in the appropriate tab

3. **Users can create wallets** - New currency immediately available for wallet creation

4. **Type is automatic** - Wallet type determined from currency type

## Exchange Rate Data

### Sources (In Priority Order)

1. **Database (pairs table)** - Pre-populated via seed script
   - 25+ fiat pairs (USD to all fiat currencies)
   - 40+ crypto pairs (all cryptos to USD, PHP, etc.)

2. **OpenExchangeRates API** (fallback)
   - Updates hourly
   - ~160+ currencies supported
   - Requires API key (already configured)

3. **CoinGecko API** (crypto fallback)
   - Free, no API key needed
   - 10,000+ cryptocurrencies
   - Limited to 10-50 calls/minute

4. **ExConvert API** (last resort)
   - Supports 31 cryptos, 163 fiat currencies
   - Requires API key (already configured)

### Caching
- Rates cached for **1 hour**
- Fallback to cache if APIs fail
- Manual refresh available via `/functions/v1/fetch-rates`

## Troubleshooting

### Currency Not Showing in Modal
- **Check:** Is it in the currencies table?
- **Check:** Is `active = true`?
- **Check:** Does it have a `type` set (fiat or crypto)?
- **Solution:** Run migration 058 to ensure currencies are created

### Wallet Created But No Rate Showing
- **Check:** Are rates populated in `pairs` table?
- **Check:** Is edge function deployed?
- **Solution:** Run `supabase functions deploy fetch-rates`

### Only Seeing Fiat Currencies
- **Check:** Are crypto currencies in database?
- **Check:** Do they have `type = 'crypto'`?
- **Solution:** Check migrations 058 and 059
